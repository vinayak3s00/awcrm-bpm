DAY 1-2: REPORTING DATABASE SCHEMA
===================================

TASK 5.1: Reporting Database Design
-----------------------------------
□ Extend PostgreSQL schema in src/models/Schema.ts:

REPORTS TABLE:
```sql
- id (UUID, primary key)
- name (varchar, not null)
- description (text)
- report_type (enum: standard, custom, scheduled, dashboard)
- category (enum: sales, marketing, activities, performance, executive)
- config (jsonb, report configuration)
- filters (jsonb, default filters)
- chart_config (jsonb, visualization settings)
- is_public (boolean, default false)
- is_favorite (boolean, default false)
- created_by (UUID, foreign key to users)
- shared_with (UUID[], array of user IDs)
- organization_id (UUID, foreign key)
- created_at (timestamp)
- updated_at (timestamp)
```

REPORT_SCHEDULES TABLE:
```sql
- id (UUID, primary key)
- report_id (UUID, foreign key to reports)
- schedule_type (enum: daily, weekly, monthly, quarterly)
- schedule_config (jsonb, cron-like configuration)
- recipients (UUID[], array of user IDs)
- email_recipients (text[], array of email addresses)
- format (enum: pdf, excel, csv, html)
- is_active (boolean, default true)
- last_sent_at (timestamp)
- next_send_at (timestamp)
- organization_id (UUID, foreign key)
- created_at (timestamp)
```

ANALYTICS_CACHE TABLE:
```sql
- id (UUID, primary key)
- cache_key (varchar, unique)
- data (jsonb)
- expires_at (timestamp)
- organization_id (UUID, foreign key)
- created_at (timestamp)
```

KPI_DEFINITIONS TABLE:
```sql
- id (UUID, primary key)
- name (varchar, not null)
- description (text)
- calculation_method (text)
- target_value (decimal)
- unit (varchar)
- category (varchar)
- is_active (boolean, default true)
- organization_id (UUID, foreign key)
- created_at (timestamp)
- updated_at (timestamp)
```

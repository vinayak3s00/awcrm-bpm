DAY 1-2: LEAD DATABASE SCHEMA
==============================

TASK 2.1: Lead Database Design
------------------------------
□ Extend PostgreSQL schema in src/models/Schema.ts:

LEADS TABLE:
```sql
- id (UUID, primary key)
- title (varchar, not null)
- description (text)
- contact_id (UUID, foreign key to contacts)
- company_name (varchar)
- industry (varchar)
- lead_source (enum: website, referral, social, email, phone, event, other)
- status (enum: new, contacted, qualified, unqualified, converted, lost)
- score (integer, 0-100)
- priority (enum: low, medium, high, urgent)
- estimated_value (decimal)
- probability (integer, 0-100)
- expected_close_date (date)
- actual_close_date (date)
- assigned_to (UUID, foreign key to users)
- tags (text[])
- custom_fields (jsonb)
- notes (text)
- created_by (UUID, foreign key to users)
- organization_id (UUID, foreign key)
- created_at (timestamp)
- updated_at (timestamp)
```

LEAD_ACTIVITIES TABLE:
```sql
- id (UUID, primary key)
- lead_id (UUID, foreign key to leads)
- activity_type (enum: call, email, meeting, note, status_change, score_change)
- subject (varchar)
- description (text)
- duration_minutes (integer)
- outcome (varchar)
- next_action (varchar)
- scheduled_at (timestamp)
- completed_at (timestamp)
- created_by (UUID, foreign key to users)
- organization_id (UUID, foreign key)
- created_at (timestamp)
```

LEAD_SCORING_RULES TABLE:
```sql
- id (UUID, primary key)
- name (varchar, not null)
- description (text)
- condition_field (varchar)
- condition_operator (enum: equals, contains, greater_than, less_than, in_list)
- condition_value (varchar)
- score_change (integer)
- is_active (boolean, default true)
- organization_id (UUID, foreign key)
- created_at (timestamp)
- updated_at (timestamp)
```

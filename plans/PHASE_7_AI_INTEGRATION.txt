DAY 1-2: AI INFRASTRUCTURE SETUP
=================================

TASK 7.1: AI Database Schema
----------------------------
□ Extend PostgreSQL schema in src/models/Schema.ts:

AI_MODELS TABLE:
```sql
- id (UUID, primary key)
- name (varchar, not null)
- description (text)
- model_type (enum: classification, regression, nlp, recommendation, prediction)
- model_config (jsonb, model configuration)
- training_data_source (varchar)
- accuracy_score (decimal)
- last_trained_at (timestamp)
- is_active (boolean, default true)
- version (varchar)
- organization_id (UUID, foreign key)
- created_at (timestamp)
- updated_at (timestamp)
```

AI_PREDICTIONS TABLE:
```sql
- id (UUID, primary key)
- model_id (UUID, foreign key to ai_models)
- entity_type (enum: lead, contact, deal, activity)
- entity_id (UUID)
- prediction_type (varchar)
- prediction_value (jsonb)
- confidence_score (decimal)
- actual_outcome (jsonb, nullable)
- is_accurate (boolean, nullable)
- created_at (timestamp)
- organization_id (UUID, foreign key)
```

AI_TRAINING_DATA TABLE:
```sql
- id (UUID, primary key)
- model_id (UUID, foreign key to ai_models)
- input_data (jsonb)
- expected_output (jsonb)
- data_source (varchar)
- quality_score (decimal)
- is_validated (boolean, default false)
- organization_id (UUID, foreign key)
- created_at (timestamp)
```

CHATBOT_CONVERSATIONS TABLE:
```sql
- id (UUID, primary key)
- session_id (varchar, not null)
- user_id (UUID, foreign key to users, nullable)
- contact_id (UUID, foreign key to contacts, nullable)
- conversation_data (jsonb)
- status (enum: active, completed, escalated, abandoned)
- satisfaction_score (integer, 1-5, nullable)
- started_at (timestamp)
- ended_at (timestamp)
- organization_id (UUID, foreign key)
```

DAY 1-2: WORKFLOW DATABASE SCHEMA
==================================

TASK 6.1: Workflow Database Design
----------------------------------
□ Extend PostgreSQL schema in src/models/Schema.ts:

WORKFLOWS TABLE:
```sql
- id (UUID, primary key)
- name (varchar, not null)
- description (text)
- category (enum: sales, marketing, support, operations, custom)
- trigger_type (enum: manual, scheduled, event_based, webhook)
- trigger_config (jsonb, trigger configuration)
- workflow_definition (jsonb, workflow steps and logic)
- is_active (boolean, default true)
- is_template (boolean, default false)
- version (integer, default 1)
- execution_count (integer, default 0)
- success_rate (decimal, default 0)
- created_by (UUID, foreign key to users)
- organization_id (UUID, foreign key)
- created_at (timestamp)
- updated_at (timestamp)
```

WORKFLOW_EXECUTIONS TABLE:
```sql
- id (UUID, primary key)
- workflow_id (UUID, foreign key to workflows)
- trigger_data (jsonb, data that triggered the workflow)
- status (enum: pending, running, completed, failed, cancelled)
- current_step (integer, default 0)
- execution_log (jsonb, step-by-step execution log)
- error_message (text)
- started_at (timestamp)
- completed_at (timestamp)
- duration_seconds (integer)
- organization_id (UUID, foreign key)
- created_at (timestamp)
```

WORKFLOW_STEPS TABLE:
```sql
- id (UUID, primary key)
- workflow_id (UUID, foreign key to workflows)
- step_order (integer, not null)
- step_type (enum: action, condition, delay, approval, notification, integration)
- step_config (jsonb, step configuration)
- conditions (jsonb, conditional logic)
- is_active (boolean, default true)
- created_at (timestamp)
```

WORKFLOW_TRIGGERS TABLE:
```sql
- id (UUID, primary key)
- name (varchar, not null)
- description (text)
- trigger_type (enum: contact_created, lead_updated, deal_moved, activity_completed, time_based, webhook)
- event_config (jsonb, event configuration)
- is_active (boolean, default true)
- organization_id (UUID, foreign key)
- created_at (timestamp)
```

WORKFLOW_TEMPLATES TABLE:
```sql
- id (UUID, primary key)
- name (varchar, not null)
- description (text)
- category (varchar)
- template_data (jsonb, workflow template definition)
- usage_count (integer, default 0)
- rating (decimal, default 0)
- is_public (boolean, default false)
- created_by (UUID, foreign key to users)
- organization_id (UUID, foreign key)
- created_at (timestamp)
```

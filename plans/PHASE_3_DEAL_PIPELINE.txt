DAY 1-2: DEAL DATABASE SCHEMA
==============================

TASK 3.1: Deal Database Design
------------------------------
□ Extend PostgreSQL schema in src/models/Schema.ts:

DEALS TABLE:
```sql
- id (UUID, primary key)
- title (varchar, not null)
- description (text)
- contact_id (UUID, foreign key to contacts)
- lead_id (UUID, foreign key to leads, nullable)
- company_name (varchar)
- deal_value (decimal, not null)
- currency (varchar, default 'USD')
- stage_id (UUID, foreign key to deal_stages)
- probability (integer, 0-100)
- expected_close_date (date)
- actual_close_date (date)
- deal_source (enum: inbound, outbound, referral, partner, event, other)
- priority (enum: low, medium, high, urgent)
- status (enum: open, won, lost, on_hold)
- assigned_to (UUID, foreign key to users)
- team_members (UUID[], array of user IDs)
- tags (text[])
- custom_fields (jsonb)
- notes (text)
- created_by (UUID, foreign key to users)
- organization_id (UUID, foreign key)
- created_at (timestamp)
- updated_at (timestamp)
```

DEAL_STAGES TABLE:
```sql
- id (UUID, primary key)
- name (varchar, not null)
- description (text)
- order_index (integer)
- probability_default (integer, 0-100)
- color (varchar)
- is_active (boolean, default true)
- is_closed_won (boolean, default false)
- is_closed_lost (boolean, default false)
- organization_id (UUID, foreign key)
- created_at (timestamp)
- updated_at (timestamp)
```

DEAL_ACTIVITIES TABLE:
```sql
- id (UUID, primary key)
- deal_id (UUID, foreign key to deals)
- activity_type (enum: call, email, meeting, note, stage_change, value_change, demo, proposal)
- subject (varchar)
- description (text)
- duration_minutes (integer)
- outcome (varchar)
- next_action (varchar)
- scheduled_at (timestamp)
- completed_at (timestamp)
- created_by (UUID, foreign key to users)
- organization_id (UUID, foreign key)
- created_at (timestamp)
```

DEAL_PRODUCTS TABLE:
```sql
- id (UUID, primary key)
- deal_id (UUID, foreign key to deals)
- product_name (varchar, not null)
- description (text)
- quantity (integer, default 1)
- unit_price (decimal)
- total_price (decimal)
- discount_percent (decimal, default 0)
- created_at (timestamp)
```

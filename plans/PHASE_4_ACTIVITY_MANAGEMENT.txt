DAY 1-2: ACTIVITY DATABASE SCHEMA
==================================

TASK 4.1: Activity Database Design
----------------------------------
□ Extend PostgreSQL schema in src/models/Schema.ts:

ACTIVITIES TABLE (Enhanced):
```sql
- id (UUID, primary key)
- type (enum: call, email, meeting, task, note, demo, proposal, follow_up, other)
- subject (varchar, not null)
- description (text)
- status (enum: scheduled, in_progress, completed, cancelled, overdue)
- priority (enum: low, medium, high, urgent)
- duration_minutes (integer)
- outcome (varchar)
- next_action (varchar)
- scheduled_at (timestamp)
- completed_at (timestamp)
- due_date (timestamp)
- reminder_at (timestamp)
- location (varchar)
- meeting_url (varchar)
- contact_id (UUID, foreign key to contacts, nullable)
- lead_id (UUID, foreign key to leads, nullable)
- deal_id (UUID, foreign key to deals, nullable)
- assigned_to (UUID, foreign key to users)
- created_by (UUID, foreign key to users)
- participants (UUID[], array of user IDs)
- tags (text[])
- custom_fields (jsonb)
- organization_id (UUID, foreign key)
- created_at (timestamp)
- updated_at (timestamp)
```

ACTIVITY_TEMPLATES TABLE:
```sql
- id (UUID, primary key)
- name (varchar, not null)
- description (text)
- activity_type (varchar)
- default_duration (integer)
- default_subject (varchar)
- default_description (text)
- is_active (boolean, default true)
- organization_id (UUID, foreign key)
- created_by (UUID, foreign key to users)
- created_at (timestamp)
```

COMMUNICATION_LOGS TABLE:
```sql
- id (UUID, primary key)
- activity_id (UUID, foreign key to activities)
- communication_type (enum: email, sms, call, video_call, chat)
- direction (enum: inbound, outbound)
- from_address (varchar)
- to_address (varchar)
- subject (varchar)
- content (text)
- status (enum: sent, delivered, opened, clicked, bounced, failed)
- external_id (varchar)
- metadata (jsonb)
- organization_id (UUID, foreign key)
- created_at (timestamp)
```

CALENDAR_INTEGRATIONS TABLE:
```sql
- id (UUID, primary key)
- user_id (UUID, foreign key to users)
- provider (enum: google, outlook, apple, other)
- external_calendar_id (varchar)
- access_token (text, encrypted)
- refresh_token (text, encrypted)
- sync_enabled (boolean, default true)
- last_sync_at (timestamp)
- organization_id (UUID, foreign key)
- created_at (timestamp)
- updated_at (timestamp)
```
